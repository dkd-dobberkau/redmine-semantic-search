<task type="auto">
  <name>Task 2: Implement config system with viper YAML + env overrides and fail-fast validation</name>
  <files>
    internal/config/config.go
    config.example.yml
    .env.example
  </files>
  <action>
Create `internal/config/config.go` with:

1. A `Config` struct using `mapstructure` tags and `validate` tags:
   - `RedmineURL` (string, `validate:"required,url"`, mapstructure: `redmine_url`)
   - `RedmineAPIKey` (string, `validate:"required"`, mapstructure: `redmine_api_key`)
   - `QdrantHost` (string, `validate:"required"`, mapstructure: `qdrant_host`)
   - `QdrantPort` (int, default 6334, mapstructure: `qdrant_port`)
   - `EmbeddingURL` (string, `validate:"required,url"`, mapstructure: `embedding_url`)

2. A `Load() (*Config, error)` function that:
   - Sets config name to "config", type "yaml", path "."
   - Sets defaults for ALL fields (including empty strings) — critical for viper AutomaticEnv + Unmarshal to work (see RESEARCH pitfall #1)
   - Calls `viper.SetEnvKeyReplacer(strings.NewReplacer(".", "_"))` then `viper.AutomaticEnv()`
   - Reads config file (tolerates ConfigFileNotFoundError — env-only is valid)
   - Unmarshals to `Config` struct
   - Validates with `validator.New(validator.WithRequiredStructEnabled())`
   - On validation failure, returns error with all missing fields listed

Create `config.example.yml` with full comments explaining every option, defaults, and env var override names:
```yaml
# Redmine connection
redmine_url: "https://redmine.example.com"  # Required. Env: REDMINE_URL
redmine_api_key: ""                          # Required. Env: REDMINE_API_KEY
# Qdrant connection
qdrant_host: "localhost"                     # Required. Env: QDRANT_HOST
qdrant_port: 6334                            # Default: 6334 (gRPC). Env: QDRANT_PORT
# Embedding service
embedding_url: "http://localhost:8080"       # Required. Env: EMBEDDING_URL
```

Create `.env.example` listing all env vars with empty/default values as a quick-start reference.

**CRITICAL per research:** Every field MUST have a `viper.SetDefault()` call (even empty string for required fields). Without this, `AutomaticEnv()` + `Unmarshal()` silently ignores environment variables in viper v1.19+.
  </action>
