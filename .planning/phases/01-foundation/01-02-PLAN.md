Create `deployments/Dockerfile` as a multi-stage build per research pattern:

**Stage 1 — builder:**
```dockerfile
FROM golang:1.23-alpine AS builder
WORKDIR /app
RUN apk add --no-cache git ca-certificates
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -o /app/indexer ./cmd/indexer
```

**Stage 2 — runtime:**
```dockerfile
FROM alpine:3.20
RUN apk add --no-cache ca-certificates curl && \
    adduser -D -u 1000 appuser
WORKDIR /app
COPY --from=builder /app/indexer .
COPY --chown=appuser:appuser config.example.yml ./config.example.yml
USER appuser
EXPOSE 8090
HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
    CMD curl -f http://localhost:8090/health || exit 1
CMD ["./indexer"]
```

Note: The health check endpoint `/health` will be implemented in Phase 2. For Phase 1, the health check will fail gracefully (container still runs, just not "healthy" — acceptable for Phase 1 which focuses on collection init and benchmark, not a running server).
