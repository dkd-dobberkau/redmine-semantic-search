**Create `bench/recall/testdata.go`** with 50+ synthetic QA pairs:

Define a `QAPair` struct with `Query string` and `Passage string`. Create a `var BenchmarkPairs = []QAPair{...}` slice with at minimum:

- 20 German-language pairs resembling Redmine issues:
  - Issue titles as queries, issue descriptions as passages
  - Mix: bug reports, feature requests, support tickets
  - Include short queries (3-5 words) and longer queries
  - Example: Query: "Fehler beim Upload von Anhängen", Passage: "Beim Hochladen von Anhängen größer als 10MB tritt ein Timeout-Fehler auf. Der Fehler betrifft alle Projekte und ist reproduzierbar mit Firefox und Chrome. Logs zeigen einen 504 Gateway Timeout."

- 20 English-language pairs:
  - Same variety: bugs, features, support
  - Example: Query: "login fails with LDAP", Passage: "Authentication via LDAP fails when the domain controller is unreachable. Error: connection timeout after 5s. Affects all users in the EU office. Workaround: use local authentication."

- 10 cross-lingual or edge-case pairs:
  - Short code-related queries: "NullPointerException bei PDF-Export"
  - Queries with version numbers: "Update auf Version 4.2 bricht Datenbankmigrationen"
  - Wiki-style passages (longer, more structured)

Passages should NOT include the `"passage: "` prefix — the Embedder handles that. Each passage should be 2-5 sentences of realistic Redmine content.

**Create `bench/recall/main.go`** as a standalone program (not `go test`):

1. Parse flags:
   - `--qdrant-host` (default: localhost)
   - `--qdrant-port` (default: 6334)
   - `--embedding-url` (default: http://localhost:8080)
   - `--threshold` (default: 0.75, float64)
   - `--k` (default: 10, int)
   - `--no-prefix` (default: false — run WITHOUT prefixes for comparison)

2. Create TEI embedder and Qdrant client.

3. Create a temporary Qdrant collection (name: `bench_recall_temp_{timestamp}`) with:
   - Same vector params as production (768d, Cosine)
   - No payload indexes needed (benchmark only)
   - Clean up (delete collection) at the end via `defer`

4. Embed all passages with `EmbedPassages()` and upsert to the temp collection.
   - Use sequential integer IDs (0, 1, 2, ...) as point IDs — no UUID needed for benchmark
   - Wait for upsert to complete

5. For each QA pair:
   - Embed the query with `EmbedQuery()`
   - Search top-K in the temp collection (no filters)
   - Check if the correct passage index is in the top-K results
   - Track hits

6. Compute and print results:
   ```
   === Recall@10 Benchmark ===
   Model: multilingual-e5-base (768d)
   Prefixes: enabled (passage:/query:)
   Pairs: 50
   Hits: 45/50
   Recall@10: 0.9000
   Threshold: 0.75
   Result: PASS
   ```

7. Also compute and print MRR@10 (Mean Reciprocal Rank) as a secondary metric.

8. Exit with code 0 on pass, code 1 on fail.

9. **Retry logic for TEI cold start:** The benchmark should retry the first embedding call with exponential backoff (using `cenkalti/backoff/v4`) up to 2 minutes. TEI may still be loading the model when the benchmark starts. After the first successful call, no more retries needed.

**When `--no-prefix` is set:** Create a special embedder wrapper that calls the TEI `/embed` endpoint directly WITHOUT adding `"passage: "` / `"query: "` prefixes. This proves the prefix delta. The benchmark should be run twice (with and without prefixes) and both results documented.
