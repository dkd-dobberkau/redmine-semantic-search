Create `internal/indexer/sync.go` in package `indexer`:

**Syncer struct:**
```go
type Syncer struct {
    redmine  *redmine.Client
    pipeline *Pipeline
    interval time.Duration
    batch    int           // max issues per poll cycle (from config SyncBatchSize)
    cursor   time.Time     // updated_on cursor — advances after each successful batch
    logger   *slog.Logger
    cancel   context.CancelFunc
    done     chan struct{}
}
```

**Constructor:**
`NewSyncer(redmine *redmine.Client, pipeline *Pipeline, interval time.Duration, batchSize int, logger *slog.Logger) *Syncer`
- Initialize cursor to `time.Time{}` (zero value = epoch, which means first run fetches everything from the beginning — per decision "start with empty index, begin incremental polling immediately").
- Initialize done channel.

**Start(ctx context.Context):**
- Create derived context with cancel: `ctx, cancel := context.WithCancel(ctx)`. Store cancel.
- Launch goroutine that:
  1. Runs `s.poll(ctx)` immediately on start (first poll).
  2. Creates `time.NewTicker(s.interval)`.
  3. Loops: select on ticker.C (call poll) or ctx.Done (break).
  4. On exit, close(s.done).
- Non-blocking — returns immediately.

**Stop():**
- Call cancel(), then `<-s.done` to wait for goroutine exit.

**poll(ctx context.Context):**
1. Log `logger.Info("sync: polling", "since", s.cursor)`
2. Call `s.redmine.FetchIssuesSince(ctx, s.cursor, 0, s.batch)` to fetch one page.
3. If err != nil and ctx is canceled, return silently. Otherwise log error and return (retry next cycle).
4. If len(issueList.Issues) == 0, log "sync: no updates" and return.
5. Call `s.pipeline.IndexIssues(ctx, issueList.Issues)`.
6. If err != nil, log error and return (cursor NOT advanced — retry next cycle).
7. Advance cursor: find the max `updated_on` from the fetched issues (parse RFC3339 timestamps, take the latest). Set `s.cursor = maxUpdatedOn`.
8. Log `logger.Info("sync: indexed", "count", len(issueList.Issues), "cursor", s.cursor)`
9. If `issueList.TotalCount > issueList.Offset + len(issueList.Issues)`, log that more pages remain — they will be picked up next cycle (bounded page approach per user decision).

CRITICAL per user decisions:
- Bounded pages per cycle (config batch size, default 100). Do NOT paginate through all pages in one cycle.
- Service stays responsive — poll is non-blocking relative to HTTP server.
- Cursor advances monotonically — only after successful indexing.

**Create `internal/indexer/reconcile.go`:**

`Reconciler` struct:
```go
type Reconciler struct {
    redmine *redmine.Client
    qdrant  *qdrant.Client
    cron    *cron.Cron
    logger  *slog.Logger
}
```

`NewReconciler(redmine *redmine.Client, qdrantClient *qdrant.Client, schedule string, logger *slog.Logger) (*Reconciler, error)`:
- Create `cron.New()` from `robfig/cron/v3`.
- Add function job with the schedule expression. If invalid cron expression, return error.
- Return reconciler (cron NOT started yet — caller calls Start).

`Start()`: calls `r.cron.Start()`.
`Stop()`: calls `r.cron.Stop()` and waits (returns context from Stop).

`reconcile(ctx context.Context)` — the job function:
1. Log `logger.Info("reconcile: starting ID diff")`
2. Fetch all Redmine issue IDs via `r.redmine.FetchAllIssueIDs(ctx)`. Build a `map[int64]struct{}` set.
3. Scroll all Qdrant points with `content_type=issue`, fetching only `redmine_id` payload. Use cursor-based pagination with `ScrollPoints`:
   - Collection: `qdrant.AliasName`
   - Filter: Must = `[qdrant.NewMatch("content_type", "issue")]`
   - Limit: 1000 per page
   - WithPayload: include only `redmine_id`
   - WithVectors: false
   - Use `client.Scroll(ctx, &qdrant.ScrollPoints{...})` — returns `([]*qdrant.RetrievedPoint, *qdrant.PointId)`. Loop until next offset is nil (Pitfall 3: cursor-based, NOT integer offset).
4. For each Qdrant point, extract `redmine_id` from payload. If NOT in the Redmine ID set, it is an orphan.
5. Collect orphan Qdrant point IDs (as `*qdrant.PointId`).
6. If orphans exist, batch-delete in groups of 500 using `client.Delete(ctx, &qdrant.DeletePoints{...})` with a `PointsSelector` of point UUIDs.
7. Log `logger.Info("reconcile: complete", "orphans_deleted", orphanCount, "total_qdrant", qdrantCount, "total_redmine", redmineCount)`
