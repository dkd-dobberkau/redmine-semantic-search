  <action>
Create `internal/indexer/pipeline.go` in package `indexer` with:

**Dependencies injected via struct:**
```go
type Pipeline struct {
    embedder embedder.Embedder
    qdrant   *qdrant.Client
    logger   *slog.Logger
}
```
- `NewPipeline(embedder embedder.Embedder, qdrantClient *qdrant.Client, logger *slog.Logger) *Pipeline` constructor.

**Core function `IndexIssues(ctx context.Context, issues []redmine.Issue) error`:**
1. For each issue, build the embeddable text: `issue.Subject + "\n\n" + text.StripMarkup(issue.Description)`. If Description is empty, use Subject alone.
2. Call `text.ChunkText(fullText)` to split into overlapping chunks.
3. Before upserting, call `DeleteIssueChunks(ctx, issue.ID)` to remove any existing chunks for this issue. This handles re-indexing when chunk count changes (Pitfall 4 from research).
4. Collect all chunks across all issues into a flat slice of `chunkEntry` (struct with issueRef, chunkIndex, chunkTotal, text).
5. Build chunk texts slice and call `p.embedder.EmbedPassages(ctx, chunkTexts)` — the TEI embedder already handles batch-32 chunking internally.
6. Build `[]*qdrant.PointStruct` from embeddings. For each chunk:
   - Point ID: use a new helper `ChunkPointID(redmineID int, chunkIndex int) string` that calls `uuid.NewSHA1(qdrant.PointIDNamespace, []byte(fmt.Sprintf("issue:%d:chunk:%d", redmineID, chunkIndex))).String()`. Place this helper in the same file (not in qdrant/pointid.go to avoid circular imports — indexer imports qdrant package already).
   - Actually, to avoid issues, define `ChunkPointID` in `internal/qdrant/pointid.go` by adding a new exported function there. The indexer already imports `internal/qdrant`. No circular dependency.
   - Payload via `qdrant.NewValueMap(map[string]any{...})`:
     - `"redmine_id"`: issue.ID (int)
     - `"content_type"`: `"issue"` (string)
     - `"project_id"`: issue.Project.ID (int)
     - `"tracker"`: issue.Tracker.Name (string)
     - `"status"`: issue.Status.Name (string)
     - `"author"`: issue.Author.Name (string)
     - `"subject"`: issue.Subject (string)
     - `"is_private"`: issue.IsPrivate (bool)
     - `"text_preview"`: first 500 characters of the chunk text (for snippet display)
     - `"chunk_index"`: chunkIndex (int)
     - `"chunk_total"`: len(chunks) (int)
     - `"created_on"`: issue.CreatedOn (string — RFC3339 from Redmine)
     - `"updated_on"`: issue.UpdatedOn (string — RFC3339 from Redmine)
   - Vector: `qdrant.NewVectors(embedding...)`
   - ID: `qdrant.NewIDUUID(chunkPointID)`
7. Batch upsert points. Use batch size of 100 points per upsert call (Qdrant handles large batches well but we bound for memory). Set `Wait: true` for durability.
8. Log: `logger.Info("indexed issues", "count", len(issues), "chunks", totalChunks)`

**Helper `DeleteIssueChunks(ctx context.Context, redmineID int) error`:**
- Delete all existing points for this issue using a Qdrant filter on `content_type=issue` AND `redmine_id=<id>`.
- Use `p.qdrant.Delete(ctx, &qdrant.DeletePoints{...})` with `PointsSelector` set to a filter selector. Research Pattern 6 shows the filter approach. Construct:
  ```go
  &qdrant.DeletePoints{
      CollectionName: qdrantpkg.AliasName,
      Points: &qdrant.PointsSelector{
          PointsSelectorOneOf: &qdrant.PointsSelector_Filter{
              Filter: &qdrant.Filter{
                  Must: []*qdrant.Condition{
                      qdrant.NewMatch("content_type", "issue"),
                      qdrant.NewMatchInt("redmine_id", int64(redmineID)),
                  },
              },
          },
      },
  }
  ```
- If go-client doesn't support this pattern directly, fall back to the simpler approach: use `qdrant.NewPointsSelectorFilter` if it exists, or construct the protobuf manually as shown above.
- Note: The `Wait` field should be set to `true` (&trueVal) to ensure deletion completes before new upsert.

**Also add `ChunkPointID` to `internal/qdrant/pointid.go`:**
- Add `func ChunkPointID(redmineID, chunkIndex int) string` that uses `uuid.NewSHA1(PointIDNamespace, []byte(fmt.Sprintf("issue:%d:chunk:%d", redmineID, chunkIndex))).String()`. This extends the existing PointID pattern.

**Helper `truncate(s string, maxLen int) string`** — returns first maxLen runes of s, used for text_preview payload.
