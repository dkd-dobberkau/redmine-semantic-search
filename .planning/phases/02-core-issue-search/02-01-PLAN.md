Create `internal/redmine/models.go` with Go structs matching Redmine JSON API response shapes:
- `Issue` struct: ID (int), Subject (string), Description (string), Project (IDRef), Tracker (IDRef), Status (IDRef), Priority (IDRef), Author (IDRef), AssignedTo (*IDRef), IsPrivate (bool), CreatedOn (string), UpdatedOn (string). Use `json:"snake_case"` tags matching Redmine JSON keys (`id`, `subject`, `description`, `project`, `tracker`, `status`, `priority`, `author`, `assigned_to`, `is_private`, `created_on`, `updated_on`).
- `IDRef` struct: ID (int), Name (string) — used for all Redmine references (project, tracker, status, author, etc.).
- `IssueList` struct: Issues ([]Issue), TotalCount (int), Offset (int), Limit (int) — matching Redmine's pagination envelope (`issues`, `total_count`, `offset`, `limit` JSON keys).
- `User` struct: ID (int), Login (string), FirstName (string), LastName (string), Mail (string), Admin (bool), Memberships ([]Membership) — for `/users/current.json?include=memberships` response.
- `Membership` struct: ID (int), Project (IDRef), Roles ([]IDRef).
- `UserResponse` struct: User (User) — Redmine wraps user in `{"user": {...}}`.
- `Project` struct: ID (int), Identifier (string), Name (string).
- `ProjectList` struct: Projects ([]Project), TotalCount (int), Offset (int), Limit (int).

Create `internal/redmine/client.go` with:
- `Client` struct holding baseURL (string), httpClient (*http.Client with 10s timeout), and apiKey (string).
- `NewClient(baseURL, apiKey string) *Client` constructor that trims trailing slash from baseURL.
- `doJSON(ctx context.Context, apiKey string, path string, params url.Values, target any) error` — builds full URL from baseURL+path+"?"+params.Encode(), creates http.NewRequestWithContext GET, sets `X-Redmine-API-Key` header to the provided apiKey (NOT c.apiKey — the apiKey parameter allows callers to use either the admin key or a user key), decodes JSON into target. Returns sentinel errors:
  - `var ErrUnauthorized = errors.New("redmine: unauthorized")` — returned on HTTP 401/403
  - `var ErrNotFound = errors.New("redmine: not found")` — returned on HTTP 404
  - Wraps other non-2xx as `fmt.Errorf("redmine: unexpected status %d", resp.StatusCode)`
- `doJSONWithAdminKey(ctx, path, params, target)` — convenience that calls `doJSON` with `c.apiKey` (the admin key configured at startup).
- `GetCurrentUser(ctx context.Context, apiKey string) (*User, error)` — calls `GET /users/current.json?include=memberships` using the provided user API key (NOT admin key). Decodes into UserResponse, returns &resp.User.
- `ListProjects(ctx context.Context, apiKey string) ([]Project, error)` — paginates through `GET /projects.json?limit=100&offset=N` using the provided user API key. Loops until all projects are fetched (offset >= total_count). Returns the full slice.

Create `internal/redmine/issues.go` with:
- `FetchIssuesSince(ctx context.Context, since time.Time, offset, limit int) (*IssueList, error)` — builds params with `url.Values`: `updated_on` set to `">="+since.UTC().Format(time.RFC3339)`, `status_id` set to `"*"`, `sort` set to `"updated_on:asc"`, `limit` and `offset` as strings. Calls `doJSONWithAdminKey`. The `url.Values.Encode()` handles URL-encoding of `>=` automatically (Pitfall 1 from research). Returns decoded IssueList.
- `FetchAllIssueIDs(ctx context.Context) ([]int, error)` — paginates through `GET /issues.json?status_id=*&sort=id:asc&limit=100&offset=N` using admin key. Returns all issue IDs as a sorted int slice. Used for deletion reconciliation. Fetches only `id` by not requesting includes.

CRITICAL per research:
- Always pass `status_id=*` to include closed issues (Pitfall 2).
- Use `url.Values.Set()` which auto-encodes the `>=` operator (Pitfall 1).
- `sort=updated_on:asc` for cursor-based incremental fetch to advance monotonically.

Create `internal/text/strip.go`:
- Package `text` with a `StripMarkup(text string) string` function.
- Define compiled regexps at package level (var block) for Textile and Markdown patterns:
  - `rePre` — `<pre>[\s\S]*?</pre>` removes preformatted blocks (replace with single space)
  - `reHTMLTag` — `<[^>]+>` removes any HTML tag (replace with single space)
  - `reHeader` — `(?m)^h[1-6]\.\s+` removes Textile headers (replace with empty)
  - `reBold` — `\*(\S[^*]*\S)\*` extracts bold content (keep inner text via `$1`)
  - `reItalic` — `_(\S[^_]*\S)_` extracts italic content (keep inner text via `$1`)
  - `reStrike` — `-(\S[^-]*\S)-` extracts strikethrough content (keep inner text via `$1`)
  - `reCode` — `@([^@]+)@` extracts inline code (keep inner text via `$1`)
  - `reTextileLink` — `"([^"]+)":(https?://\S+)` extracts Textile link text (keep `$1`)
  - `reMarkdownLink` — `\[([^\]]+)\]\([^)]+\)` extracts Markdown link text (keep `$1`)
  - `reMarkdownImage` — `!\[([^\]]*)\]\([^)]+\)` removes Markdown images (replace with `$1` or empty)
  - `reMultiSpace` — `\s{2,}` normalizes whitespace to single space
- Apply regexps in order: pre blocks first, then HTML tags, then headers, then inline formatting, then links, then whitespace normalization. TrimSpace at the end.
- Handle empty input by returning empty string.

Create `internal/text/chunk.go`:
- Constants: `ChunkSize = 1600` (characters, ~400 tokens), `ChunkOverlap = 200` (characters, ~50 tokens). Per research discretion recommendations.
- `ChunkText(text string) []string` function:
  - Convert to `[]rune` for correct Unicode handling (Redmine content is DE/EN mixed).
  - If `len(runes) <= ChunkSize`, return `[]string{text}` (single chunk, no splitting needed).
  - Sliding window: start at 0, end at start+ChunkSize (capped at len(runes)). Append `string(runes[start:end])`. If end == len(runes), break. Otherwise advance start to `end - ChunkOverlap`.
  - Return slice of chunk strings.
- Each chunk is a self-contained text passage ready for embedding. Callers combine StripMarkup + ChunkText.
