<objective>
Build the authentication middleware and permission resolution cache that enforce Redmine access control on every search request.

Purpose: Without auth, the search API would expose all indexed content to any caller. This plan implements the two-phase permission model: pre-filtering (project_ids injected into Qdrant filter) and the data structures needed for post-filtering (is_private check). The permission cache prevents Redmine from being hammered on every search request.

Output: `internal/auth/middleware.go` (HTTP middleware) and `internal/auth/permissions.go` (permission cache with TTL and singleflight).
</objective>

Create `internal/auth/permissions.go` in package `auth`:

**UserPermissions struct:**
```go
type UserPermissions struct {
    UserID     int
    Login      string
    IsAdmin    bool
    ProjectIDs []int64  // int64 for direct use in Qdrant filter (NewMatchInt takes int64)
}
```

**PermissionCache struct:**
```go
type PermissionCache struct {
    mu      sync.RWMutex
    entries map[string]cacheEntry  // api_key -> cached permissions
    sf      singleflight.Group
    ttl     time.Duration
    redmine *redmine.Client
    logger  *slog.Logger
}

type cacheEntry struct {
    perms     *UserPermissions
    expiresAt time.Time
}
```

**NewPermissionCache(redmineClient *redmine.Client, ttl time.Duration, logger *slog.Logger) *PermissionCache:**
- Initialize entries map.

**Resolve(ctx context.Context, apiKey string) (*UserPermissions, error):**
1. Read lock: check if apiKey exists in entries and `time.Now().Before(e.expiresAt)`. If so, return cached perms.
2. Cache miss: use `sf.Do(apiKey, func() (any, error) { return c.fetchFromRedmine(ctx, apiKey) })` to coalesce concurrent requests.
3. If err, return it (sentinel errors from Redmine client propagate: ErrUnauthorized for 401/403, wrapped error for network failure).
4. Write lock: store result with `expiresAt = time.Now().Add(c.ttl)`.
5. Return perms.

**fetchFromRedmine(ctx context.Context, apiKey string) (*UserPermissions, error):**
1. Call `c.redmine.GetCurrentUser(ctx, apiKey)` — validates the key and returns user identity.
   - If `redmine.ErrUnauthorized`, return it immediately (do NOT cache invalid keys).
   - If other error (network), return wrapped error.
2. Call `c.redmine.ListProjects(ctx, apiKey)` — returns all projects the user can access.
   - If error, return wrapped error.
3. Extract project IDs as `[]int64`.
4. Build and return `&UserPermissions{UserID: user.ID, Login: user.Login, IsAdmin: user.Admin, ProjectIDs: projectIDs}`.

**Invalidate(apiKey string):**
- Write lock, delete entry. (Useful for future cache-busting, not called in Phase 2 but good to have.)

CRITICAL per research:
- singleflight.Group prevents cache stampede on concurrent requests with the same key.
- TTL is short (5 minutes default from config) — security trade-off is acceptable per user decision.
- Invalid keys are NOT cached — only successful lookups.
- ErrUnauthorized propagates for 401 response; other errors propagate for 503 response.

Create `internal/auth/middleware.go` in package `auth`:

**Context key type and exported accessor:**
```go
type contextKey string
const ctxUserKey contextKey = "user"

// UserFromContext extracts the authenticated UserPermissions from the request context.
// Returns nil if no user is set (should never happen behind AuthMiddleware).
func UserFromContext(ctx context.Context) *UserPermissions {
    u, _ := ctx.Value(ctxUserKey).(*UserPermissions)
    return u
}
```

**AuthMiddleware struct:**
```go
type AuthMiddleware struct {
    cache  *PermissionCache
    logger *slog.Logger
}
```

**NewAuthMiddleware(cache *PermissionCache, logger *slog.Logger) *AuthMiddleware** constructor.

**Wrap(next http.Handler) http.Handler:**
Returns `http.HandlerFunc` that:
1. Extract API key: `apiKey := r.Header.Get("X-Redmine-API-Key")`.
2. If empty, respond with `{"error": "missing X-Redmine-API-Key header"}`, status 401. Return.
3. Call `m.cache.Resolve(r.Context(), apiKey)`.
4. If error is `redmine.ErrUnauthorized`:
   - Respond with `{"error": "invalid API key"}`, status 401. Return.
5. If error is any other (Redmine unreachable):
   - Log error at Warn level.
   - Respond with `{"error": "authentication service unavailable"}`, status 503. Return.
6. On success: inject UserPermissions into context via `r.WithContext(context.WithValue(r.Context(), ctxUserKey, perms))`.
7. Call `next.ServeHTTP(w, r.WithContext(...))`.

**JSON error responses:** Set `Content-Type: application/json` header before writing. Use `json.NewEncoder(w).Encode(map[string]string{"error": msg})` or manual fmt.Fprintf for simple structures. Ensure `w.WriteHeader(statusCode)` is called BEFORE writing body.

**Error classification using `errors.Is`:**
- `errors.Is(err, redmine.ErrUnauthorized)` for 401.
- All other non-nil errors are 503 (Redmine unreachable or internal).

Per user decisions:
- Header name is `X-Redmine-API-Key` (matches Redmine's own convention).
- 401 for invalid/missing key, 503 for Redmine unreachable.
- UserPermissions in context includes ProjectIDs for pre-filter and IsAdmin for potential future use.
