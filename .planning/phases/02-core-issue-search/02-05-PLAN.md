**Create `internal/search/handler.go` in package `search`:**

**SearchHandler struct:**
```go
type SearchHandler struct {
    embedder embedder.Embedder
    qdrant   *qdrant.Client
    logger   *slog.Logger
}
```

**NewSearchHandler(embedder embedder.Embedder, qdrantClient *qdrant.Client, logger *slog.Logger) *SearchHandler** constructor.

**ServeHTTP(w http.ResponseWriter, r *http.Request):**

1. **Parse query parameters:**
   - `q` (required): search query text. If empty, return 400 `{"error": "missing required parameter: q"}`.
   - `page` (optional, default 1): `strconv.Atoi(r.URL.Query().Get("page"))`. Min 1.
   - `per_page` (optional, default 20): `strconv.Atoi(r.URL.Query().Get("per_page"))`. Min 1, max 100. Clamp to bounds.
   - Filter params (all optional, pass through to Qdrant filter): `tracker`, `status`, `project`, `author`.

2. **Get user permissions from context:**
   `user := auth.UserFromContext(r.Context())`. If nil (should not happen behind middleware), return 500.

3. **Build Qdrant permission pre-filter:**
   - Build a `*qdrant.Filter` with `Must` containing a single condition: `project_id` IN user.ProjectIDs.
   - Use `Filter.Should` with N `qdrant.NewMatchInt("project_id", id)` conditions wrapped in a `Must` filter (per research Pattern 4).
   - If additional filter params provided (tracker, status, project, author), add them as additional `Must` conditions using `qdrant.NewMatch("tracker", value)` etc.

4. **Embed the query:**
   `queryVec, err := h.embedder.EmbedQuery(r.Context(), q)`. If err, return 500.

5. **Query Qdrant with oversampling:**
   - Oversampling factor: 2 (fetch 2x results for post-filtering headroom).
   - Calculate: `fetchLimit := uint64(page * perPage * 2)` — fetch enough to fill the requested page after post-filtering.
   - Call `h.qdrant.Query(ctx, &qdrant.QueryPoints{...})` with:
     - CollectionName: `qdrantpkg.AliasName`
     - Query: `qdrant.NewQueryDense(queryVec)`
     - Filter: the built permission+extra filter
     - Limit: fetchLimit
     - Offset: nil (always fetch from top, post-filter handles pagination)
     - WithPayload: `qdrant.NewWithPayloadInclude("redmine_id", "subject", "tracker", "status", "project_id", "author", "is_private", "text_preview", "chunk_index", "chunk_total")`
   - Handle error: return 500.

6. **Post-filter private issues:**
   - For each result, check `is_private` payload field.
   - If true AND user.IsAdmin is false AND author != user.Login, exclude the result.
   - This implements AUTH-03 oversampling post-filter.

7. **Deduplicate multi-chunk results:**
   - Group by `redmine_id`. For each issue, keep only the chunk with the highest score.
   - This ensures each issue appears at most once in results.

8. **Paginate:**
   - After dedup, slice the results: `start = (page-1) * perPage`, `end = min(start+perPage, len(results))`.
   - If start > len(results), return empty results slice.

9. **Build response JSON:**
   ```go
   type SearchResponse struct {
       Results    []SearchResult `json:"results"`
       Total      int            `json:"total"`        // total deduped results (approximation)
       Page       int            `json:"page"`
       PerPage    int            `json:"per_page"`
       Facets     *FacetResponse `json:"facets"`
   }
   type SearchResult struct {
       IssueID    int     `json:"issue_id"`
       Subject    string  `json:"subject"`
       Score      float32 `json:"score"`
       Snippet    string  `json:"snippet"`
       Tracker    string  `json:"tracker,omitempty"`
       Status     string  `json:"status,omitempty"`
       ProjectID  int     `json:"project_id"`
       Author     string  `json:"author,omitempty"`
   }
   ```
   - Snippet: first 150 characters of `text_preview` payload field (per discretion recommendation).
   - Score: from `result.Score` (Qdrant QueryPoints returns score).

10. **Fetch facets in parallel:**
    Call `FetchFacets(ctx, h.qdrant, permissionFilter)` (from facets.go) — runs concurrently with step 9 building. Include facets in response.

11. Set `Content-Type: application/json`, encode response with `json.NewEncoder(w).Encode(resp)`.

**Helper functions:**
- `extractPayloadString(point, key) string` — safely extract string from payload map.
- `extractPayloadInt(point, key) int` — safely extract int from payload map.
- `extractPayloadBool(point, key) bool` — safely extract bool from payload map.
- `truncateSnippet(s string, maxLen int) string` — first maxLen runes.
- `buildPermissionFilter(projectIDs []int64, tracker, status, project, author string) *qdrant.Filter` — builds the combined pre-filter.

**Create `internal/search/facets.go`:**

**FacetResponse struct:**

```go
type FacetResponse struct {
    Trackers []FacetValue `json:"trackers"`
    Statuses []FacetValue `json:"statuses"`
    Projects []FacetValue `json:"projects"`
    Authors  []FacetValue `json:"authors"`
}
type FacetValue struct {
    Value string `json:"value"`
    Count int    `json:"count"`
}
```

**FetchFacets(ctx context.Context, client *qdrant.Client, filter *qdrant.Filter) (*FacetResponse, error):**
1. Use `sync.WaitGroup` or `errgroup.Group` (from `golang.org/x/sync/errgroup`, already available since we add `golang.org/x/sync` in 02-04) to run four Qdrant Facet calls concurrently:
   - `client.Facet(ctx, &qdrant.FacetCounts{CollectionName: qdrantpkg.AliasName, Key: "tracker", Filter: filter, Limit: qdrant.PtrOf(uint64(50)), Exact: qdrant.PtrOf(false)})`
   - Same for "status", "project_id", "author".
2. CRITICAL: Pass the same permission filter to each Facet call (Pitfall 6 from research).
3. Convert `[]*qdrant.FacetHit` to `[]FacetValue` for each dimension.
4. Return assembled `FacetResponse`.

Note on `client.Facet` API: The go-client Facet method returns `([]*qdrant.FacetHit, error)`. Each FacetHit has a `Value` (FacetValue union — could be string or integer) and `Count` (uint64). Extract the string representation for the JSON response.

**ServeHTTP(w http.ResponseWriter, r *http.Request):**
1. Check Qdrant: call `h.qdrant.HealthCheck(ctx)` or `h.qdrant.CollectionExists(ctx, qdrantpkg.AliasName)` — whichever is available in the go-client. If the client has a `HealthCheck` method, use it. Otherwise, use `CollectionExists` as a liveness probe.

2. Check TEI embedding service: make an HTTP GET to `h.embeddingURL + "/health"` with a 5-second timeout. Check for 200 status.
   - Status: "ok" or "error" with message.
3. Build response:
   ```go
   type HealthResponse struct {
       Status    string          `json:"status"`    // "ok" if all checks pass, "degraded" otherwise
       Qdrant    ComponentHealth `json:"qdrant"`
       Embedding ComponentHealth `json:"embedding"`
   }
   type ComponentHealth struct {
       Status  string `json:"status"`  // "ok" or "error"
       Message string `json:"message,omitempty"`
   }
   ```
4. Overall status: "ok" if both components are ok, "degraded" if any component has error.
5. HTTP status code: 200 if overall ok, 503 if degraded.
6. Set `Content-Type: application/json`, encode response.

**Create `cmd/server/main.go`:**

The HTTP search server entrypoint:

1. Load config via `config.Load()`. Fatal on error.
2. Set up `slog.Logger` with JSON handler to stdout.
3. Connect to Qdrant: create gRPC client. Defer close. Follow the same pattern as in cmd/indexer/main.go.
4. Create TEI embedder: `embedder.NewTEIEmbedder(cfg.EmbeddingURL)`.
5. Create Redmine client: `redmine.NewClient(cfg.RedmineURL, cfg.RedmineAPIKey)` — needed for permission cache.
6. Create permission cache: `auth.NewPermissionCache(redmineClient, time.Duration(cfg.PermissionCacheTTL)*time.Minute, logger)`.
7. Create auth middleware: `auth.NewAuthMiddleware(permCache, logger)`.
8. Create search handler: `search.NewSearchHandler(teiEmbedder, qdrantClient, logger)`.
9. Create health handler: `search.NewHealthHandler(qdrantClient, cfg.EmbeddingURL, logger)`.
10. Set up routing with Go 1.22 `net/http.ServeMux`:
    ```go
    mux := http.NewServeMux()
    mux.Handle("GET /api/v1/search", authMiddleware.Wrap(searchHandler))
    mux.HandleFunc("GET /api/v1/health", healthHandler.ServeHTTP)
    ```
    - Search endpoint requires auth (wrapped with middleware).
    - Health endpoint is public (no auth — needed for Docker health checks and monitoring).
11. Create `http.Server` with:
    - Addr: `cfg.ListenAddr` (default ":8090")
    - Handler: mux
    - ReadTimeout: 10 * time.Second
    - WriteTimeout: 30 * time.Second
12. Start server in goroutine: `go srv.ListenAndServe()`.
13. Log: `logger.Info("search server started", "addr", cfg.ListenAddr)`
14. Wait for OS signal (SIGINT, SIGTERM) using `signal.NotifyContext`.
15. On signal: call `srv.Shutdown(shutdownCtx)` with a 15-second deadline. Log shutdown status.
